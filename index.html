База данных

База данных — совокупность самостоятельных материалов, систематизированных таким образом, чтобы эти материалы могли быть найдены и обработаны с помощью ЭВМ.

  • Реляционная
  • Сетевая
  • Функциональная

Реляционная БД

Или таблично-ориентированная БД

Любые данные из реляционной БД могут быть представлены в виде таблицы.

Каждая запись в таблице имеет идентификатор (id).

Операторы SQL

  • DDL — Data Definition Language (операторы определения данных)
  • DML — Data Manipulation Language (операторы манипуляции данными)
  • DCL — Data Control Language (операторы определения доступа к данным)
  • TCL — Transaction Control Language (операторы управления транзакциями)

DML

Data Manipulation Language (операторы манипуляции данными)

INSERT

Позволяет добавлять новые записи в таблицу

INSERT INTO <таблица> (<колонка1>, <колонка2>) VALUES (<значение1>, <зачение2>);

Есть и другие варианты синтаксиса, могут зависеть от СУБД.

Не все поля обязательны.

Строчные приняты, но не обязательны.

Форматирование

Важно!

INSERT
INTO <таблица> (<колонка1>, <колонка2>)
VALUES (<значение1>, <зачение2>) ;


INSERT
INTO users (firstname, lastname, age)
VALUES ('Сергей', 'Козлов', 24);

INSERT

Позволяет изменять записи в таблице

UPDATE [top(x)] <объект>
SET <присваивание1> [, присваивание2, ...]
[WHERE <условие>];

Пример:

UPDATE tbl_books
SET price = 0
WHERE quantity = 0;

DELETE

Позволяет удалять записи из таблицы

DELETE
FROM <таблица>
WHERE <условие>;

Пример:

DELETE
FROM users
WHERE age < 18 OR firstname IS NULL;

SELECT

Позволяет получить проекцию из таблиц

SELECT * FROM users;

SELECT login FROM users;

SELECT firstname, lastname FROM users;

SELECT login FROM users WHERE age IS NULL;

SELECT * FROM users ORDER BY date_register DESC;

SELECT * FROM users LIMIT 60, 30;

DDL

Data Definition Language (операторы определения данных)

Позволяет разрешать или запрещать выполнять те или иные операции пользователям.

  • CREATE — создает объект БД
  • ALTER — изменяет объект
  • DROP — удаляет объект

DCL

Data Control Language (операторы определения доступа к данным)

Позволяет разрешать или запрещать выполнять те или иные операции пользователям.

  • GRANT — предоставляет пользователю (группе) разрешения на определенные операции с объектом
  • REVOKE — отзывает ранее выданные разрешения
  • DENY — задает запрет, имеющий приоритет над разрешением
